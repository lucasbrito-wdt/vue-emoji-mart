# @vue-emoji-mart/data

[![License: MIT](https://img.shields.io/badge/license-MIT-blue.svg)](../../LICENSE)

Emoji datasets and i18n locale files consumed by [`@vue-emoji-mart/core`](../vue-emoji-mart). Framework-agnostic: it's plain JSON generated by a Node build script, with no dependency on Vue.

## Install

```bash
pnpm add @vue-emoji-mart/data
```

## Usage

### Default import (native set)

```ts
import data from '@vue-emoji-mart/data'
```

### A specific emoji set

```ts
import appleData from '@vue-emoji-mart/data/sets/15/apple.json'
// Available sets: native, apple, facebook, google, twitter
```

### A locale file

```ts
import pt from '@vue-emoji-mart/data/i18n/pt.json'
```

Available locales: `ar`, `be`, `cs`, `de`, `en`, `es`, `fa`, `fi`, `fr`, `hi`, `it`, `ja`, `ko`, `nl`, `pl`, `pt`, `ru`, `sa`, `tr`, `uk`, `vi`, `zh`. English (`en`) is built into `@vue-emoji-mart/core` and doesn't need to be imported explicitly.

## Contents

```
data/
├── sets/{version}/{set}.json   # emoji datasets, one file per emoji version × rendering set
├── i18n/{locale}.json          # UI translations
├── index.d.ts                  # TypeScript types for the dataset schema
└── build.js                    # regenerates sets/ and index.d.ts from emoji-datasource
```

## Build

```bash
pnpm --filter @vue-emoji-mart/data build
```

Regenerates the JSON datasets and type definitions. The output schema matches [`@emoji-mart/data`](https://github.com/missive/emoji-mart/tree/main/packages/emoji-mart-data) — this package is a self-hosted port, not a wrapper, so upstream schema updates need to be merged manually into `build.js`.
